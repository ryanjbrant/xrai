## Detailed Profiles

### Mark Weiser (1952-1999)
**Domain**: Calm Computing, Ubiquitous Computing
**Affiliation**: Xerox PARC

**Core Contributions**:
- Coined "ubiquitous computing" (1988)
- Invented "calm technology" concept
- Center-to-periphery attention model

**Key Papers**:
- "The Computer for the 21st Century" (Scientific American, 1991)
- "Designing Calm Technology" (with John Seely Brown, 1996)

**Resources**:
- [Calm Technology Design Patterns](https://calmtech.com/)
- [Xerox PARC Ubiquitous Computing](https://www.parc.com/research/past/ubicomp/)

**KB Cross-References**:
- `_XRAI_FORMAT_SPECIFICATION_V2.md` → Ambient data layers
- `_WEB_INTEROPERABILITY_STANDARDS.md` → Peripheral awareness

**Design Principles Extracted**:
```
1. Information moves between center and periphery
2. Peripheral attunement without explicit attention
3. Technology informs without demanding focus
```

### Ted Nelson
**Domain**: Hypertext, Document Architecture
**Affiliation**: Project Xanadu (1960-present)

**Core Contributions**:
- Coined "hypertext" and "hypermedia" (1963)
- Project Xanadu - original hypertext vision
- Transclusion - quoting with visible source
- ZigZag - multidimensional data structure
- Two-way links with provenance

**Key Books**:
- "Computer Lib / Dream Machines" (1974)
- "Literary Machines" (1981)

**Resources**:
- [Project Xanadu](https://xanadu.com/)
- [Ted Nelson Homepage](https://thenelsons.org/ted/)
- [ZigZag](http://xanadu.com/zigzag/)

**KB Cross-References**:
- `_XRAI_HYPERGRAPH_WORLD_GENERATION.md` → Provenance tracking
- `_WEB_INTEROPERABILITY_STANDARDS.md` → Two-way links
- `_XRAI_FORMAT_SPECIFICATION_V2.md` → Transclusion model

**Design Principles Extracted**:
```
1. Every quote should link to its source (transclusion)
2. Links should be bidirectional
3. Provenance is always visible
4. Documents are living, evolving structures
5. "Everything is deeply intertwingled"
```
